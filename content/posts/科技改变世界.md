## 从 jQuery 到 Vue：一段前端简史

2006 年，jQuery 发布。它的口号是 "Write Less, Do More"——在那个浏览器 API 残缺不全的年代，`$('.btn').fadeOut()` 一行代码就是魔法。

二十年过去了。

今天我们要面对的是 Vite 的 esbuild 预编译、Tree Shaking 死代码消除、Suspense 异步边界、Hydration 水合……前端开发变成了一个高度工程化的领域。

## 三个阶段

### 1. DOM 操作时代（2006–2013）

```
浏览器兼容性 → jQuery → 面条式代码 → 难以维护
```

这个时期的特征是**直接操作 DOM**。每个交互都是一串选择器、回调、样式修改。页面复杂度低的时候一切都很美好，但当交互超过十个，代码开始失控。

典型问题：一个按钮的 `<a href="#"></a>`

### 2. MV* 框架时代（2013–2018）

AngularJS 带来了双向绑定，React 带来了虚拟 DOM，Vue 把两者的优点打包成渐进式方案。

核心转变是**数据驱动视图**。你不再直接操作 DOM，而是修改数据，框架负责把数据映射到界面。

```javascript
// 不再这样写
document.getElementById('count').innerText = n

// 而是这样
const count = ref(0)
```

这个思维转变比任何具体 API 都重要。它意味着你把"怎么做"（命令式）换成了"是什么"（声明式）。

### 3. 工程化时代（2018–至今）

工具链的进化是这个阶段的主旋律：

- **打包**：Webpack → Rollup → esbuild → Rolldown
- **语言**：JavaScript → TypeScript（类型安全成为标配）
- **运行时**：纯 CSR → SSR → SSG → ISR → Edge Rendering
- **样式**：CSS → SCSS → CSS-in-JS → Tailwind → 原子化 CSS

一个有意思的现象是**循环往复**。SSR 其实就是十年前的 PHP 模板渲染，只不过现在用 JavaScript 做。Tailwind 的 `class="flex items-center gap-4"` 和当年直接写 inline style 异曲同工。区别在于，现在的回归是带着工程经验的——不是回到原点，而是螺旋上升。

## 为什么轻量级技术栈值得关注

这个博客就是一个例子：

| 层面 | 选择 |
|------|------|
| 框架 | Vue 3 |
| 构建 | Vite（几乎零配置） |
| 内容 | Markdown 文件 |
| 部署 | 纯静态 HTML |

没有数据库，没有后端服务，没有 CI/CD 管道里跑十分钟的构建——整个站从源码到上线不到一秒。

"重"不是原罪，"不必要的重"才是。如果你维护一个十个页面的公司官网，你真的需要 React Server Components 吗？

## 写在最后

> 工具应该匹配问题，而不是反过来。

前端技术仍在快速演进——2025 年我们看到 Rolldown（Rust 写的打包器）、React 的 Server Functions、Vue Vapor Mode（无虚拟 DOM 模式）。但选择技术栈的标准始终应该是：

1. 解决当前问题
2. 团队能驾驭
3. 未来两年内不会成为负债

不必追每个新框架，但要理解每个新想法背后的动机。那些动机里藏着下一个十年的方向。
